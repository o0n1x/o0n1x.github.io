Network Control System

Spring 2025 | 12 Weeks | Individual Project

A sophisticated network traffic management pipeline enabling precise control of latency and bandwidth for distributed systems in a federated learning environment.

Python Linux Traffic Control RabbitMQ Distributed Systems

Project Overview

This project's goal is to develop dynamic network control for a federated learning testbed (CoLExT) where multiple devices with varied architectures need to communicate efficiently while simulating real-world network conditions.

20+ Devices

Managed network features across diverse Linux devices with varied architectures

1000+ Lines of Code

Developed efficient system integrate seamlessly with existing infrastructure

5+ Test Scripts

Self-developed testing suite ensuring reliability of each pipeline component

Problem Statement

Federated learning Research and Development needs an environment where real world scenarios can be tested on algorithms, complete network control is essential in simulating these environments, but comes with various challenges:

Challenge 1: Heterogeneity

the system needs to support multiple environments and extreme scenarios due to the nature of federated learning heterogeneity in connected devices.

Challenge 2: Dynamic Control

Testbed users need to dynamically adjust network parameters (latency, bandwidth, packet loss) during experiments to simulate real-world conditions like poor connectivity or network congestion.

Challenge 3: Concurrency

the system is a distributed network of devices each is working in its own separate timeline, so concurrency needs to be taken into account across different devices and network scenarios

Solution Approach

Architecture Design

The system follows a distributed architecture pattern with centralized control and decentralized execution:

Core Components:
  • Control Server: Central management system that orchestrates network configurations and hosts Message broker
  • Publish/Subscribe Message Queue (RabbitMQ): Asynchronous communication layer for synchronization and communication
  • Device Agents: Lightweight agents running on each device including server to execute network control commands
  • YAML Translator: Sub system that translates input config YAML file into usable tc commands for each device

Implementation Details

Automated Pipeline

Developed Python scripts that automate from single YAML file to complete network control management both statically and dynamically for each device given the input file using Kubernetes

Message Queue Integration

Implemented RabbitMQ for reliable, time synchronization for dynamic network commands

Seamless and modular Integration

Designed the system as a standalone system that works alone if needed and made each core component as a module that work alone and only communicate once with other component if possible

Technical Implementation

# Example: Network Control Command Input Structure
                            
network:
  - tag: default
    upstream:
      bandwidth: 50Mbps
    downstream:
      bandwidth: 50Mbps
  - tag: DynFast
    dynamic:
    - iterator: time 
      structure: [bandwidth, latency, delay-distribution] 
      commands:
      - [20, set, outgoing, 1000Mbps, 0.1ms, normal]
      - [100, del, outgoing]
      - [100, set, outgoing, 1000Mbps, 0.1ms, normal]
      - [120, set, outgoing, 10Mbps,  0.1ms, normal]
clients:
  - dev_type: OrangePi5B
    count: 2
    network: 
    - default
    - DynFast

In this example, we have set 2 network configurations (default and DynFast) where the first is static and second is dynamic respectively, and the OrangePi client is set to use both networks. the static network is applied before the test starts and the dynamic network is applied overtime according to the configuration set.

Key Features

Static and Dynamic Network Control

Adjust bandwidth, latency, jitter, and packet loss parameters in real-time across all connected devices through a unified config file.

Time synchronization for dynamic control

time stamps used in dynamic network commands are synced across devices for reliable test results and metrics

Monitoring & Logging

Comprehensive logging system tracks all network changes and provides real-time monitoring of network conditions.

Scalable Design

Can easily scale from a few devices to hundreds without architectural changes.

Modular and Usable Design

system coded with modularity in mind, making it easy to maintain and build upon in the future

Technologies & Tools

Programming Languages

Python 3.9+: Primary language for system implementation, chosen for its excellent system integration capabilities and extensive library support.

Bash Scripting: Used for system-level operations and automation scripts.

Message Queue

RabbitMQ: Enterprise-grade message broker for reliable asynchronous communication between components, ensuring command delivery even during network disruptions.

Network Control

Linux Traffic Control (TC): Kernel-level network traffic management tool for controlling bandwidth, latency, and packet loss.

Other Tools

Kubernetes Config Maps a Kubernetes object designed to store non-confidential configuration data as key-value pairs and mounted to containers

Logging: Python logging module with custom formatters for comprehensive system monitoring.

Lessons Learned

Technical Insights

  • Understanding how Network Control (TC) works at a low level is crucial for effective traffic management
  • Message queues like RabbitMQ are powerful tools for building resilient distributed systems
  • Using python to execute complex system calls is often a more effective method instead of using a Bash script with Python

Project Management

  • Early and frequent testing saves significant debugging time later
  • Documentation should be developed concurrently with development for more accuracy
  • Modular design allows for easier testing and maintenance and future development
  • understanding the project requirements deeply at the start can significantly improve speed and quality
