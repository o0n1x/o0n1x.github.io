Project Overview

Onyx Notes is a personal project built for developers and terminal users who want to take notes directly in the terminal with a nice TUI and intuitive shortcuts. It brings vault-based organization and Markdown editing to the terminal with a clean, modern TUI.

Multiple Vaults

Create, delete, and manage multiple vaults to organize notes by project, topic, or context

Keyboard Driven

Efficient navigation throughout the app with intuitive shortcuts for every action

Multi-Panel Interface

Edit, preview, and view statistics simultaneously with modular panel management

Motive

I built this to get hands-on experience with TUI development, file I/O patterns, and shipping a complete application from design to release. Along the way, several engineering perspectives were reinforced:

User Experience Design

TUI applications demand intuitive design thinking. When building a tool meant for frequent, rapid use, any friction in the interaction flow significantly impacts the overall experience. This project sharpened my ability to anticipate user needs and minimize cognitive load.

File and Vault Organization

Managing note files across multiple vaults required evaluating different storage strategies and determining which approach best serves the application's purpose—balancing simplicity, performance, and portability.

Full Application Lifecycle

This project provided experience across the complete development cycle: from initial design and planning, through implementation and feature iteration, to user testing and preparing a release.

Key Features

Multiple Vaults

Create and manage separate vaults for different contexts. Quick search on startup lets you filter and select vaults instantly.

Nested Folder Structure

Organize notes in hierarchical directories. Create notes with paths like folder/subfolder/note.md and folders are created automatically.

Modular Panel Management

Toggle panels independently with F1 (Editor), F2 (Viewer), F3 (Metadata) to customize your workspace.

Note Metadata & Statistics

Real-time word count, line count, creation date, tags, and modification timestamps displayed in the metadata panel.

YAML Frontmatter Support

Similar to the Obsidian note-taking app, Onyx Notes includes metadata (title, tags, created/modified dates) in YAML frontmatter.

Technical Implementation

Architecture

The application is structured with clear separation of concerns:

Project Structure:
onyx-notes/
├── src/
│   ├── __init__.py    # Package init
│   ├── tui.py         # Main TUI application (Textual)
│   ├── note.py        # Note data model
│   ├── vault.py       # Vault management
│   ├── note_io.py     # File I/O operations
│   ├── graph.py       # Planned feature
│   └── style.tcss     # Textual CSS styling
├── requirements.txt   # Dependencies
├── start.sh           # Linux/Mac launcher
├── start.bat          # Windows launcher
└── README.md          # Documentation

Note Format

Notes are stored as Markdown files with YAML frontmatter, maintaining compatibility with Obsidian:

---
title: My Note
tags: [example, demo]
created: 2025-01-15 10:30:00
last_modified: 2025-01-15 14:20:00
---

# My Note

Note content goes here in Markdown format...

Key Design Decisions

Textual Framework

Chose Textual for its modern and simple approach to TUI development in Python, with CSS-like styling and reactive components.

Frontmatter Library

Uses the frontmatter library for parsing YAML metadata, ensuring compatibility with existing Markdown tools while keeping it simple.

File-Based Storage

Notes are plain Markdown files on disk, making backup, sync, and external editing straightforward.

Portable Distribution

Includes portable release with start/installation scripts for all platforms, no installation required.
